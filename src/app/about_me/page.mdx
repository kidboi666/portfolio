## 빠른 학습력과 검증된 생산성

“항상 학습 진도가 빠르고 학습에 오랜 시간을 투자하는 게 느껴집니다.”

이 문장은 부트캠프 동료에게 실제로 들은 피드백입니다. 저는 흥미를 느끼는 분야에 깊이 파고드는 성향으로, 2024년 1월 유튜브 영상 하나로 시작한 개발이 현재는 실제 서비스 운영으로 이어졌습니다.

부트캠프 6개월간 진행한 3개의 팀 프로젝트에서 평균 40-46%의 기여도를 기록했습니다(팀원 4-5명, 커밋 수 기준). 개념 학습 후 빠르게 프로젝트에 적용하며 시행착오를 직접 경험하는 방식으로, '아이디어를 웹 화면으로 구현'하는 프로그래밍의 본질적인 즐거움을 느낄 수 있었습니다.

---

## 프로젝트를 완주하는 실행력

2025년 2월부터 6개월간 MoodLog라는 감정 기록 앱을 1인 개발했습니다. React Native로 시작했으나 성능 이슈로 Flutter로 전환을 결정했고, 2개월 만에 32,000줄의 코드를 재작성하여 Google Play Store와 App Store에 정식 출시했습니다.

기획부터 개발, QA, 앱스토어 심사 대응, 개인사업자 등록까지 서비스 런칭의 전 과정을 경험했습니다. 출시 후에도 29회의 버전 업데이트를 진행하며 4개월간 안정적으로 운영했습니다. 이 경험은 단순히 코드를 작성하는 것을 넘어, 사용자에게 실제 가치를 전달하는 서비스를 만드는 법을 배운 시간이었습니다.

---

## 아키텍처 설계 및 개선 능력

웹 개인 프로젝트 OneLog를 개발하며 초기 계층형 구조 (components, hooks, services)의 한계를 경험했습니다. 기능이 추가될수록 컴포넌트 간 의존성이 복잡해지고 재사용성이 떨어지는 문제가 발생했습니다.

이를 해결하기 위해 Feature-Sliced Design 아키텍처를 학습하고, Claude Code를 활용해 대규모 마이그레이션을 진행했습니다. 6개 레이어(app, pages, widgets, features, entities, shared)로 도메인 로직을 분리하고, 계층 간 의존성 규칙을 통해 순환 참조를 방지했습니다.

이 과정에서 '동작하는 코드'와 '확장 가능한 코드'의 차이를 이해했고, 장기적인 관점에서 유지보수 가능한 아키텍처의 중요성을 체득했습니다.

## 사용자를 배려하는 개발 태도

프론트엔드 개발자에게 가장 중요한 것은 사용자의 관점에서 생각하는 능력이라고 믿습니다.

부트캠프 프로젝트에서 팀원들이 API 응답 데이터를 확인하느라 어려움을 겪는 것을 보고, 관리자 페이지를 자발적으로 제작했습니다. API 호출 시 전달받는 데이터를 시각적으로 확인할 수 있게 만들어 팀의 개발 속도를 높였습니다.

OneLog 프로젝트에서는 사용자의 디바이스 크기에 따라 메뉴 위치가 자연스럽게 조정되도록 수십 번의 테스트를 반복했습니다. TanStack Query의 Optimistic Update를 구현해 사용자가 좋아요를 누르면 서버 응답을 기다리지 않고 즉시 UI가 반응하도록 개선했습니다.

작은 불편함도 놓치지 않으려는 이러한 태도가 결국 더 나은 사용자 경험으로 이어진다고 생각합니다.

## 끈질긴 문제 해결력

5년간의 건설 현장 경험에서 체득한 철저한 계획과 책임감은 꼼꼼한 코드 작성과 테스트를 우선시하는 개발 습관으로 이어졌습니다.

프론트엔드 개발에서 새로운 문제는 항상 예상치 못하게 나타나지만, 저는 문제를 만났을 때 포기하지 않고 끝까지 해결책을 찾습니다. 다만 팀 프로젝트에서는 효율성이 중요하기에, 일정을 고려하며 최적의 해결 방법을 찾으려 노력합니다.

Drizzle ORM으로의 마이그레이션, FSD 아키텍처 전환, Server Component 최적화 등 도전적인 과제들을 하나씩 해결하며 기술적으로 성장해왔습니다. 어려움이 클수록 성취감도 크기에, 저는 복잡한 문제를 해결하는 과정 자체를 즐깁니다.
